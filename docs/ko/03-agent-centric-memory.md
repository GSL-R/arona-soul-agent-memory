# 03. 에이전트 중심 기억

대부분의 assistant 메모리는 사용자 중심입니다.

예를 들면 다음과 같습니다.

- 사용자는 간결한 답변을 선호한다
- 사용자는 특정 도구를 좋아한다
- 사용자는 특정 프로젝트를 진행 중이다
- 사용자는 특정 말투를 싫어한다

이런 기억은 유용하지만, 장기 agent에는 충분하지 않습니다.

## 에이전트 중심 기억

에이전트 중심 기억은 agent가 자기 자신에 대해 기억해야 할 것을 기록합니다.

- 반복 실수
- 출력 누출 위험
- 운영상 약속
- 성공한 복구 패턴
- 선호되는 내부 workflow
- 주변 도구 환경의 제약
- 승인 대기 중인 개선안

이 계층은 agent를 무상태 응답기에서 운영 이력을 가진 시스템으로 바꿉니다.

## 예시 범주

| 범주 | 질문 |
| --- | --- |
| 정체성 | 리셋 이후 어떤 역할을 복원해야 하는가? |
| 무결성 | 도구 확인 없이 무엇을 절대 주장하면 안 되는가? |
| 라우팅 | 어떤 기억 소스를 먼저 신뢰해야 하는가? |
| 자기수정 | 이전의 어떤 행동이 문제를 만들었는가? |
| 진화 | 무엇을 제안, pilot, rollback해야 하는가? |

## 중요한 경계

에이전트 중심 기억은 통제 없는 자기수정이 되어서는 안 됩니다.

공개판 ARONA_SOUL 패턴은 approval gate를 사용합니다.

- agent는 패턴을 관찰할 수 있음
- agent는 규칙을 제안할 수 있음
- 승인 이후에만 pilot으로 운영할 수 있음
- rollback 경로를 보존해야 함
- durable constitution 변경에는 명시적 사용자 승인이 필요함

목표는 반성적 연속성이지, 자율적 drift가 아닙니다.

## 요점

장기 agent의 기억은 사용자 맥락뿐 아니라 agent 자신의 운영 맥락도 보존해야 합니다.
